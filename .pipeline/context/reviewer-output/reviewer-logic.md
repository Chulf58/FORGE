## Logic Review: test-author agent and pipeline split

### Issues
- [ ] **Phase-zero edge case — no test files in phase** — AC-2 line 181: Plan states "test-author first (must observe red bar), gate verification, then coder second" integrated into Phase Execution Loop, but does not specify what happens when a phase has no test files. If test-author dispatch is unconditional per-phase, the agent will have no test files to author. If dispatch is conditional (only when phase tasks mention tests), the skip logic is not stated. This ambiguity could cause the implementer to either (a) unconditionally dispatch test-author and leave it with no work, or (b) add conditional dispatch logic without specification.

- [ ] **Task 5 forward-reference — handoff spec timing** — AC-3 line 131 and Phase 2 Task 5 lines 185-188: AC-3 (pre-implementation) demands `agents/test-author.md` specify the handoff artefact format (`## Test files written`, `## Failure output`, no reasoning). But Task 5 is "Wave 3" (post-Wave 2 implementation). If Task 2 (agents/test-author.md Wave 2) does not include the spec, and Task 5 (Wave 3) writes it, there's a forward-reference: the agent frontmatter in Wave 2 will lack the spec it's supposed to enforce, leaving Wave 2 implementation incomplete. The plan should clarify that Task 2 includes a **comment or stub** for the handoff format, which Task 5 expands into full prose.

- [ ] **Test structure clarity — what fails before implementation** — AC-6 line 168 demands test cases assert "(a) the wave-split step list includes a `test-author` invocation before the coder step". However, the test does not yet exist (Phase 1 Task 1), and the plan does not detail what assertion will fail when test-author invocation logic is missing. Is this an AST check of SKILL.md? A mock dispatch check? A runtime assertion on the step list object shape? The implementer needs to know which assertion to write so it fails when the feature is absent.
